OfferOps — Automation Bible (Strict Technical Specification)
PURPOSE
This document is the SINGLE SOURCE OF TRUTH for OfferOps automation behavior.
The system MUST NOT guess logic.
Every automation, sync, task, state transition, and workflow action MUST follow this specification exactly.

SYSTEM SOURCE OF TRUTH
Voluum is source of truth for:
	•	Offers
	•	Tracker Campaigns
	•	Visits
	•	ROI
	•	Tracker campaign existence
	•	Tracker campaign stats

OfferOps App is source of truth for:
	•	Tasks
	•	Workflow states
	•	Notifications
	•	Batch progression
	•	Traffic source order
	•	User permissions
	•	Manual workflow actions
	•	Admin settings

SYNC ENGINE RULES
Sync Scope
Sync MUST ONLY:
	•	import NEW offers
	•	import NEW tracker campaigns
	•	update offer stats
	•	update tracker campaign stats
Sync MUST NOT:
	•	recreate traffic sources
	•	recreate affiliate networks
	•	recreate existing entities

Sync Buttons
Sync buttons are allowed in:
	•	Batch page
	•	Admin dashboard
	•	Workspace dashboard

OFFER IMPORT EVENT
EVENT
OfferImported

CONDITIONS
When sync imports a NEW offer.

ACTIONS
	•	Validate tag regex
	•	Parse tag
	•	Find existing batch
	•	IF batch missing:
	•	create batch
	•	attach offer
	•	generate tasks
	•	create notification
	•	ELSE:
	•	attach offer to existing batch

IMPORTANT EDGE CASE
If:
	•	batch already exists
	•	NEW offers arrive later
	•	imported_at date differs significantly
THEN:
Create HIGH PRIORITY notification:
“Unexpected offers detected for existing batch. Manual review required.”
This scenario is considered abnormal.

BATCH STATUS MACHINE
Allowed Batch Statuses ONLY
	•	NEW_BATCH
	•	WAITING_FOR_TRACKER_CAMPAIGNS
	•	OFFER_READY_FOR_LIVE_TESTING
	•	LIVE_TESTS
	•	TESTED
	•	COMPLETED

STATE TRANSITIONS
STATE 1
NEW_BATCH
Meaning:
	•	batch created
	•	no tracker campaigns imported

TRANSITION RULE
IF exactly ONE tracker campaign exists
THEN:
status = WAITING_FOR_TRACKER_CAMPAIGNS

STATE 2
WAITING_FOR_TRACKER_CAMPAIGNS
Meaning:
	•	only ios OR android exists
	•	second tracker campaign missing

REQUIRED UI BEHAVIOR
Missing task MUST:
	•	remain TODO
	•	remain flashing
	•	remain in notifications

TRANSITION RULE
IF:
	•	ios tracker campaign exists AND
	•	android tracker campaign exists
THEN:
status = OFFER_READY_FOR_LIVE_TESTING

STATE 3
OFFER_READY_FOR_LIVE_TESTING
Meaning:
	•	both tracker campaigns exist
	•	worker can start live testing

TRANSITION RULE
Worker manually changes status.
IF worker sets:
LIVE_TESTS
THEN:
status = LIVE_TESTS

STATE 4
LIVE_TESTS
Meaning:
	•	live traffic testing is active

REQUIRED UI BEHAVIOR
Batch card MUST:
	•	flash visually
	•	appear high priority

TRANSITION RULE
IF EVERY offer in batch has:
visits >= 20000
THEN:
status = TESTED

STATE 5
TESTED
Meaning:
	•	testing completed
	•	winner selection required

HARD BUSINESS RULES
RULE 1
A batch can NEVER be tested on multiple traffic sources simultaneously.

RULE 2
A batch can NEVER contain multiple affiliate networks.

RULE 3
Workers create tracker campaigns ONLY inside Voluum.
OfferOps NEVER creates Voluum campaigns automatically.

RULE 4
The app tracks workflow. The app does NOT manage traffic source campaigns directly.

ERROR HANDLING
Invalid Offer Tag
IF offer tag fails regex validation:
	•	reject automation
	•	create admin notification
	•	do NOT create batch

Invalid Tracker Campaign Tag
IF tracker campaign tag fails regex validation:
	•	reject automation
	•	create admin notification
	•	do NOT attach campaign

Duplicate Tracker Campaign
IF same device type already exists for:
	•	same batch
	•	same traffic source
THEN:
Create HIGH PRIORITY warning.

Missing Campaign Forever
IF task remains TODO for over admin-defined limit:
THEN:
	•	escalate notification
	•	keep flashing

API Failure
IF sync partially fails:
	•	rollback partial writes
	•	log error
	•	create sync warning

ADMIN SETTINGS
Traffic Source Order
Admin can:
	•	add traffic sources
	•	disable traffic sources
	•	reorder traffic sources

IMPORTANT RULE
Traffic source order changes MUST NOT affect batches already in progress.
Only NEW batches use updated order.

USER PERMISSIONS
ADMIN
Can:
	•	view all workspaces
	•	edit traffic source order
	•	manage workers
	•	trigger sync
	•	view notifications
	•	review suspicious batches

WORKER
Can:
	•	view assigned tasks
	•	update task statuses
	•	update batch workflow states
	•	view active batches
Workers CANNOT:
	•	edit traffic source order
	•	edit admin settings
	•	change sync configuration

FINAL ARCHITECTURE REQUIREMENT
The system MUST be event-driven.
The system MUST NOT contain hardcoded spaghetti automation logic.
Every automation MUST be triggered ONLY by:
	•	events
	•	rules
	•	validated states
