Build a web app called OfferOps.

This is an internal management system for an affiliate media buying team.

The app should help an admin track employees, daily reports, weekly reports, testing batches, campaign statuses, goals, and automated next-step tasks.

Tech stack:
- Use React for frontend
- Use Node.js / Express for backend
- Use PostgreSQL or Replit database
- Use simple clean professional UI
- Make it responsive
- Build authentication with roles: admin and employee

Core roles:

1. Admin
The admin can:
- See all employees
- Create and edit employees
- Set weekly and monthly goals per employee
- View daily reports
- View weekly reports
- View all testing batches
- View dashboards and KPIs
- See auto-generated tasks
- Update campaign and batch statuses
- Prepare future Voluum API integration

2. Employee
The employee can:
- Log in
- See only their own dashboard
- Submit daily reports
- Create testing batches
- Add offers to a batch
- Update task status
- See their weekly and monthly goals
- See progress toward goals
- See their assigned To-Do tasks

Important automation logic:

When a TestingBatch status changes to "tested":
1. Check the TrafficSourcePlan table for the same affiliate_network and geo.
2. Find the next traffic source by test_order that has not yet been tested.
3. If a next traffic source exists, automatically create a TodoTask for the same employee:
   Title: "Create next test batch"
   Description: "Create a new test batch for [affiliate_network] - [geo] using traffic source [next_traffic_source]. This is based on tested batch [batch_name]."
   task_type: create_next_test_batch
   status: open
   priority: high
4. If no next traffic source exists, create an admin review task:
   Title: "Testing cycle completed"
   Description: "All planned traffic sources were tested for [affiliate_network] - [geo]. Admin should review results."
   task_type: admin_review
   status: open
   priority: medium
